# 无线充电站电路数量控制安培设计

日期：2026-07-16
状态：已获用户确认

## 背景

当前无线充电站只需放入一个有效 GregTech 电路，就会直接获得固定 `16 A` 的每 Tick 供电预算。电路只控制电压等级和服务半径，无法体现更高输出能力所需的材料投入，导致早期使用成本过低、平衡性不足。

## 目标

- 电路等级继续决定充电站的输出电压、可充电物品等级和球形服务半径。
- 电路堆叠数量决定可用安培数，`1` 个电路提供 `1 A`，最多 `16` 个提供 `16 A`。
- 电路槽只允许同种电路形成一个物品堆叠，并将槽位上限强制限制为 `16`。
- 保持现有方块、MetaTileEntity ID、NBT 数据和世界存档兼容。
- 保持现有玩家优先、机器其次、安全降压、目标过滤和轮询逻辑不变。

## 交互设计

充电站仍使用一个专用电路槽。可放入的物品范围不变，仅接受以下 GregTech 矿辞电路：

| 电路 | 输出等级 | 电压 | 半径 |
| --- | --- | ---: | ---: |
| 基础电路 | LV | 32 EU/t | 16 格 |
| 良好电路 | MV | 128 EU/t | 32 格 |
| 高级电路 | HV | 512 EU/t | 64 格 |
| 数据电路 | EV | 2,048 EU/t | 128 格 |
| 精英电路 | IV | 8,192 EU/t | 256 格 |

电路槽规则：

- 单槽最大堆叠数量为 `16`。
- Minecraft 单物品堆叠模型天然要求同一槽内为同一种物品及元数据，因此不同种电路不能混合成同一个堆叠。
- 自动化和 GUI 插入都必须遵守有效电路过滤与 `16` 个上限。
- 空槽、无效物品或数量为 `0` 时，充电站停止供电。

## 能源规则

当前电路堆叠数量经过限制后作为有效安培数：

```text
有效安培 = clamp(电路数量, 0, 16)
每 Tick 总预算 = 当前电路电压 × 有效安培
内部容量 = 每 Tick 总预算 × 400
```

示例：

- `1× 高级电路`：HV，半径 64 格，预算 `512 EU/t`（`1 A`）。
- `8× 高级电路`：HV，半径 64 格，预算 `4,096 EU/t`（`8 A`）。
- `16× 高级电路`：HV，半径 64 格，预算 `8,192 EU/t`（`16 A`）。

所有目标共享同一个每 Tick 总预算：

1. 先给所有者及其 ServerUtilities 队伍在线成员的合格电动物品充电。
2. 再将剩余预算按轮询顺序供给范围内合格的 GregTech 机器。
3. 单个机器本 Tick 可请求的安培数不超过当前剩余预算折算值，也不超过有效安培数。
4. 输出电压继续限制为充电站电压与目标安全输入电压中的较小值。

## GUI 与状态显示

- 电路槽视觉位置保持不变，槽位堆叠上限改为 `16`。
- GUI 状态文本显示当前等级、有效安培、半径和总输出预算。
- 无有效电路时继续显示未配置状态。
- Waila/机器信息若已有对应输出字段，同步使用动态安培和预算，避免继续显示固定 `16 A`。

## 存档兼容性

- 不修改机器注册 ID、类名、方块类型或 NBT 字段。
- 旧存档中原有的 `1` 个电路会在更新后自然变为 `1 A`，不会丢失物品或损坏机器。
- 如外部自动化意外尝试插入超过 `16` 个电路，槽位容量负责拒绝超量部分；能源计算仍额外钳制到 `16 A`，形成防御性保护。

## 测试与验收

单元测试覆盖：

- `0`、`1`、`8`、`16` 和超过 `16` 个电路的有效安培钳制。
- 各等级电路在不同数量下的每 Tick 总预算。
- 动态内部容量仍为对应预算的 400 倍。
- 无电路或无效等级时预算和容量为零。
- 机器目标分配不会超过有效安培数。

集成级检查覆盖：

- GUI 和自动化插入上限均为 `16`。
- 单个电路运行时显示并实际输出 `1 A`。
- 16 个电路运行时恢复当前最大 `16 A` 行为。
- 玩家优先、安全降压、能源仓支持和目标排除规则保持不变。
- `clean build` 与全部现有测试通过。
